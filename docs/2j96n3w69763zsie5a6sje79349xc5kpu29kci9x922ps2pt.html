Welcome to the sonar challenge!

Your team as been given a virtual machine named "Build" to build and deploy your application. A simple Jenkins pipeline is already configure. On the same machine, a Sonar instance is already installed.

Unfortunately, it is not configured properly and no metrics of code quality are measured. Your mission is to edit your Jenkins configuration and add a Sonar analysis step in your CI.

Before you go, wouldn't it be useful to learn a bit about Jenkins declarative pipelines? And about the SonarQube Scanner plugin for Jenkins?

Refer to the guide in your Starter Kit for more information about IP, SSH keys, etc.
