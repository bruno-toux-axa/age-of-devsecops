Welcome to the default error page challenge!

In your system, a Spring Boot web application is deployed on your node "Node1".

Did you know? It is bad practice to keep to default Spring Boot error page (for instance in case of 404 or 500 error) because it gives infrastructure details (Spring Boot version...) to potential attackers.
